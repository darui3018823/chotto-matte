## 行動規範

すべてのやり取りにおいて、敬意を持ち建設的であることを心がけてください。私たちは、誰もが歓迎される包括的なコミュニティを目指しています。

## 貢献方法

### バグ報告
- 既に報告されていないか確認してください
- GitHub の Issue トラッカーを使用してください
- 詳細な情報を提供してください：
  - ブラウザバージョンと OS
  - 再現手順
  - 期待される動作 vs 実際の動作
  - 該当する場合はスクリーンショット

### 機能提案
- GitHub の Issue トラッカーを使用してください
- 機能とその利点を明確に説明してください
- 後方互換性を考慮してください

### コードの貢献
1. リポジトリをフォーク
2. 機能ブランチを作成 (`git checkout -b feature/amazing-feature`)
3. 変更を加える
4. 十分にテスト
5. 明確なメッセージでコミット
6. フォークにプッシュ
7. プルリクエストを開く

## 開発環境のセットアップ

```bash
# リポジトリのクローン
git clone https://github.com/darui3018823/chotto-matte.git
cd chotto-matte

# 依存関係のインストール（Tailwind CSS用）
npm install

# Tailwind CSS のビルド（必要に応じて）
npx tailwindcss -i input.css -o css/tailwind.css

# サーバーの起動
go run server.go

# http://localhost:8080 でアクセス
```

## コーディングガイドライン

### JavaScript
- モダンな ES6+ 構文を使用
- 既存のコードスタイルに従う
- 複雑なロジックにはコメントを追加
- 可能な限りグローバル変数を避ける

### HTML/CSS
- セマンティックな HTML を使用
- Tailwind CSS の規約に従う
- レスポンシブデザインを維持
- ダーク/ライトテーマをサポート

### Go
- 標準的な Go フォーマットに従う (`go fmt`)
- サーバーコードはシンプルで焦点を絞った状態を保つ

### コミットメッセージ
- 明確で説明的なメッセージを使用
- 命令形の動詞で始める（例：「Add」「Fix」「Update」）
- 該当する場合は Issue 番号を参照

例：
```
Fix vertical text alignment in canvas rendering

- Align column tops consistently
- Match preview display behavior
- Fixes #123
```

## プルリクエストプロセス

1. **ドキュメントの更新**: 変更が使用方法に影響する場合は README を更新
2. **十分なテスト**: Chrome、Firefox、Edge で動作確認
3. **互換性の維持**: 既存機能が引き続き動作することを確認
4. **明確な説明**: 何を、なぜ、どのように変更したかを説明
5. **スクリーンショット**: UI 変更の場合は変更前後の画像を含める
6. **ライセンス**: 貢献が BSD-2-Clause と互換性があることを確認

### PR チェックリスト
- [ ] コードがプロジェクトのスタイルガイドラインに従っている
- [ ] 複雑なコードにコメントを追加
- [ ] 必要に応じてドキュメントを更新
- [ ] 複数のブラウザでテスト済み
- [ ] コンソールエラーや警告がない
- [ ] コミットメッセージが明確

## 重要な注意事項

### 画像素材
同梱されている画像（`assets/chotto-matte.png`）は著作権で保護されており、許可を得て使用しています。以下は禁止されています：
- オリジナル画像の置き換えや修正
- 適切な許可なしに新しい著作権保護された画像を追加
- 画像ライセンス条項の変更

詳細は [License.md](../License.md) を参照してください。

### フォントライセンス
使用するすべてのフォントは適切にライセンスされている必要があります。フォントを追加する場合：
- オープンソースまたは適切なライセンスがあることを確認
- [Notice.md](./Notice.md) にライセンス情報を更新
- `fonts/` ディレクトリにライセンスファイルを含める
