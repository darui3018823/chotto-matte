# セキュリティポリシー

## 脆弱性報告

セキュリティの脆弱性を発見した場合、GitHub Issues で公開するのではなく、プロジェクト メンテナーに直接報告してください。

### 報告方法

セキュリティ上の問題を報告する場合は、以下の手順に従ってください：

1. **詳細情報を準備する**
   - 脆弱性の詳細な説明
   - 影響を受けるバージョン
   - 再現手順（可能な場合）
   - 潜在的な修正案（可能な場合）

2. **メンテナーに連絡する**
   - GitHub のセキュリティアドバイザリを通じて報告してください
   - または、プロジェクトメンテナーに直接メールで連絡してください

### レスポンスタイム

セキュリティレポートには迅速に対応することをお約束しています。通常、以下のタイムラインに従います：

- **初期応答**: 24時間以内
- **進捗更新**: 1週間以内
- **パッチ公開**: 状況に応じて

## サポートされているバージョン

| バージョン | サポート状況 |
| --- | --- |
| 最新 | ✅ セキュリティ更新サポート |
| 1つ前のメジャーバージョン | ⚠️ 重大な脆弱性のみサポート |
| それ以降 | ❌ サポート外 |

## 既知の制限事項

このプロジェクトを安全に使用するための既知の制限事項については、[README.md](README.md) および [CONTRIBUTING.md](CONTRIBUTING.md) を参照してください。

## セキュリティベストプラクティス

ユーザーの皆様は以下のセキュリティベストプラクティスに従うことをお勧めします：

- 最新バージョンにアップデートしてください
- 定期的なセキュリティ監査を実施してください
- 依存関係の脆弱性をチェックしてください

## ご質問

セキュリティに関するご質問またはご懸念がある場合は、お気軽にお問い合わせください。
