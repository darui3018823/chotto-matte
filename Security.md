# Security Policy

## Vulnerability Reporting

If you discover a security vulnerability, please do not report it through GitHub Issues. Instead, report it directly to the project maintainers.

### How to Report

If you discover a security issue, please follow these steps:

1. **Prepare detailed information**
   - Detailed description of the vulnerability
   - Affected versions
   - Steps to reproduce (if possible)
   - Potential fix (if possible)

2. **Contact the maintainers**
   - Please report through GitHub's Security Advisory
   - Or contact the project maintainers directly via email

### Response Timeline

We are committed to responding quickly to security reports. We typically follow this timeline:

- **Initial Response**: Within 24 hours
- **Progress Updates**: Within 1 week
- **Patch Release**: Depends on the situation

## Supported Versions

| Version | Support Status |
| --- | --- |
| Latest | ✅ Security update support |
| Previous major version | ⚠️ Critical vulnerabilities only |
| Earlier versions | ❌ Unsupported |

## Known Limitations

For known limitations to safely use this project, please refer to [README.md](README.md) and [CONTRIBUTING.md](CONTRIBUTING.md).

## Security Best Practices

We recommend that users follow these security best practices:

- Keep your version up to date
- Conduct regular security audits
- Check dependencies for vulnerabilities
